# 🚀 Pulse Chat — Real-time Enterprise Messenger (Slack Clone)

[![Actions Status](https://github.com/Slovuan-Swan/js-react-development-project-12/actions/workflows/hexlet-check.yml/badge.svg)](https://github.com/Slovuan-Swan/js-react-development-project-12/actions)
🌐 **Live Demo:** [Открыть демо Pulse Chat](https://pulse-chat-7r73.onrender.com)

**Pulse Chat** — это высоконагруженный SPA-мессенджер, разработанный на базе современных фронтенд-технологий. Основной фокус проекта — построение чистой архитектуры, управление сложным распределенным состоянием (Client/Server State) и обеспечение мгновенной синхронизации данных между пользователями в реальном времени.

---

## 🛠️ Архитектура и Технологический стек

- **Core:** React 19 (Hooks, Context API), TypeScript (Strict Mode)
- **Сборка статики:** Vite (Оптимизация продакшен-бандла, минимизация чанков)
- **State Management:** Redux Toolkit (Разделение на RTK Query для серверного кэша и Slices для UI-стейта)
- **Синхронизация сессий:** Redux-Persist (Автоматическое сохранение JWT-токенов в LocalStorage)
- **Real-time коммуникации:** Socket.io-client (Двухсторонний протокол WebSockets)
- **Валидация и Формы:** Formik + Yup (Асинхронная проверка уникальности каналов)
- **Интерфейс:** React-Bootstrap (Адаптивная верстка, доступность элементов по стандартам WAI-ARIA)
- **Безопасность:** Интеграция библиотеки `leo-profanity` для обсценной лексики на стороне клиента

## 🔥 Мои ключевые инженерные решения

### 1. Гибридная синхронизация данных (WebSockets + RTK Query Cache)

Вместо полной перезагрузки списков или тяжелого рендеринга всего приложения, в проекте реализована реактивная модель. Глобальный слушатель `<Sockets />` перехватывает событие сервера `newMessage` и точечно инвалидирует кэш конкретного эндпоинта. Приложение обновляет только изменившуюся область экрана, экономя трафик и ресурсы процессора.

### 2. Строгая обработка исключений (Resilience)

Приложение устойчиво к падениям:

- Интегрирован трекер ошибок **Rollbar SDK** в связке с глобальным React **ErrorBoundary**.
- Реализована каскадная обработка сетевых сбоев. Ошибки авторизации (401) и конфликты имен (409) пробрасываются напрямую в поля Formik через `setFieldError`, а обрывы связи мягко выводятся пользователю через `react-toastify`.

### 3. Модульное разделение слоев (Layered Architecture)

Код организован по строгой иерархии: `model/` (работа с данными, API и сокетами), `pages/` (высокоуровневые экраны) и `view/` (изолированные UI-компоненты и формы). Это обеспечивает слабую связность кода (Low Coupling) и позволяет масштабировать проект без проведения масштабного рефакторинга.

## 📦 Инструкция по локальному развертыванию

### Требования

- Node.js (LTS v20 или v22)
- NPM

### 1. Клонирование репозитория

```bash
git clone https://github.com/Slovuan-Swan/pulse-chat.git
cd pulse-chat
```

### 2. Установка всех зависимостей (Корень + Фронтенд)

```bash
npm install
```

### 3. Сборка продакшен-версии фронтенда

```bash
make build
```

### 4. Запуск сервера статики и API

```bash
make start
```

### 5. Открываем бэкенд-сервер

Откройте в браузере адрес: [http://localhost:5001](http://localhost:5001)
